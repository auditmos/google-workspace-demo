Findings (14)
Critical MFA nie jest wymuszone dla adminów ▶

Konta z uprawnieniami administratora nie mają wymuszonego uwierzytelniania wieloskładnikowego. Atakujący, który zdobędzie hasło admina, uzyska pełen dostęp do całej organizacji Google Workspace bez żadnej dodatkowej weryfikacji.

OpenTofu Fix
resource "googleworkspace_user" "admin" { primary_email = "admin@firma.pl" custom_schemas { schema_name = "security" schema_values = { mfa_enrolled = "true" } } } # Wymuszenie przez Admin Console policy resource "googleworkspace_org_unit" "admins" { name = "Administrators" parent_org_unit_path = "/" }
BEFORE: MFA opcjonalne → AFTER: MFA wymuszone
Critical 11 aktywnych kont byłych pracowników ▶

Zidentyfikowano 11 kont użytkowników, którzy opuścili firmę, ale ich dostępy pozostają aktywne. Konta posiadają dostęp do Google Drive, Gmail i wewnętrznych narzędzi. Stanowi to krytyczne ryzyko wycieku danych lub nieautoryzowanego dostępu.

OpenTofu Fix
# Lista aktywnych użytkowników zarządzana w kodzie # Usunięcie z kodu = automatyczna suspensja locals { active_users = toset([ "jan.kowalski@firma.pl", "anna.nowak@firma.pl", # ... tylko aktywni pracownicy ]) } resource "googleworkspace_user" "employees" { for_each = local.active_users primary_email = each.value suspended = false }
BEFORE: 11 ghost accounts → AFTER: 0 orphaned accounts
Critical Brak polityki silnych haseł ▶

Minimalna długość hasła ustawiona na 6 znaków. Brak wymogu znaków specjalnych, wielkich liter ani cyfr. Polityka nie wymusza rotacji haseł. Audyt wykazał 3 konta z hasłami krótszymi niż 8 znaków.

OpenTofu Fix
resource "googleworkspace_domain" "primary" { domain_name = "firma.pl" } # Password policy via Admin SDK # Minimum 12 znaków, complexity wymuszona resource "googleworkspace_user" "new_user" { primary_email = "user@firma.pl" password = random_password.user_pass.result change_password_at_next_login = true } resource "random_password" "user_pass" { length = 16 special = true override_special = "!@#$%" }
BEFORE: min 6 znaków → AFTER: min 12 + complexity
Critical Super Admin bez logowania audytowego ▶

Konto Super Admin nie ma włączonego alertowania o podejrzanej aktywności. Logi audytowe nie są eksportowane do zewnętrznego systemu. Brak alertów przy logowaniu z nowego urządzenia lub lokalizacji.

Rekomendacja
# Konfiguracja alert rules w Google Workspace Admin # + eksport logów do BigQuery lub SIEM resource "googleworkspace_group" "security_alerts" { email = "security-alerts@firma.pl" name = "Security Alerts" }
BEFORE: brak monitoringu → AFTER: real-time alerting
High Zewnętrzne udostępnianie Google Drive — bez ograniczeń ▶

Ustawienia Drive pozwalają użytkownikom udostępniać pliki dowolnym osobom spoza organizacji, w tym publiczne linki bez uwierzytelniania. Wykryto 47 plików udostępnionych publicznie.

OpenTofu Fix
# Ograniczenie sharing do domeny resource "googleworkspace_org_unit" "employees" { name = "Employees" parent_org_unit_path = "/" description = "Drive sharing: domain only" }
BEFORE: publiczne linki OK → AFTER: tylko @firma.pl
High Brak struktury Organizational Units ▶

Wszyscy użytkownicy (63 osoby) w jednej jednostce organizacyjnej. Brak segmentacji uniemożliwia stosowanie różnych polityk dla różnych działów. Dział finansowy i IT powinny mieć osobne, bardziej restrykcyjne polityki.

OpenTofu Fix
resource "googleworkspace_org_unit" "it" { name = "IT" parent_org_unit_path = "/" } resource "googleworkspace_org_unit" "finance" { name = "Finance" parent_org_unit_path = "/" } resource "googleworkspace_org_unit" "general" { name = "General" parent_org_unit_path = "/" }
BEFORE: 1 flat OU → AFTER: segmentacja per dział
Medium Session timeout zbyt długi (720h) ▶

Sesje użytkowników nie wygasają przez 30 dni. Rekomendowane maximum to 8-12 godzin dla zwykłych użytkowników i 1 godzina dla adminów. Długie sesje zwiększają ryzyko przejęcia konta na nieużywanym urządzeniu.

BEFORE: 720h session → AFTER: 8h / 1h admins
Medium Nieautoryzowane aplikacje OAuth z szerokim dostępem ▶

Wykryto 8 aplikacji firm trzecich z dostępem do Gmail i Drive użytkowników, z czego 3 mają scope "read all mail". Część z nich to nieużywane integracje sprzed ponad roku.

BEFORE: 8 niezaudytowanych apps → AFTER: whitelist zatwierdzonych
